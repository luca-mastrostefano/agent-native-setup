// a package manager for agent setups

Agent setups should compound.

Your contract, prompts, subagents, and guardrails install like a package and update like one. Adopt a community profile, tune it to your stack — and every improvement upstream flows into your repo, without touching what you've customized.

Browse profiles View on GitHub

// why

Agents are only as good as the repo they work in.

Claude Code, Cursor, Copilot, Gemini — all of them do their best work in a repo that carries three things, enforced by tooling rather than memory:

One contract

A single AGENTS.md every agent follows — never forking per tool.

Mechanical guardrails

Linters, hooks, and CI that catch mistakes automatically, with error messages that say how to fix them.

Feedback loops

Review subagents, tests, and an RFC trail — so quality compounds instead of drifting.

The hard part: there is no one right setup. A Python backend, a TypeScript monorepo, a legacy service — each needs different gates, prompts, and agents. Today every team re-discovers theirs from scratch, in private, and it decays. Here a setup is a profile — versioned, inspectable, forkable — and one good idea propagates to every repo that adopted it.

// quick start

One command, no install.

# brew install uv   # if you don't have uv yet — docs.astral.sh/uv
uvx --from git+https://github.com/luca-mastrostefano/agent-native-setup agent-native-setup -o ./my-app

# or scaffold from a community profile
uvx --from git+https://github.com/luca-mastrostefano/agent-native-setup agent-native-setup -o ./my-app --profile andrej-karpathy-skills-setup

Works on new and existing repos: languages are auto-detected, existing files are never overwritten, and legacy code is grandfathered so day one isn't a wall of red.

// the loop

How improvements travel.

  1. discover
  2. adopt
  3. tune
  4. snapshot or fork
  5. publish
  6. update flows back to every consumer

Every release you cut reaches every project scaffolded from your profile through agent-native-setup update — pristine files refresh, your edits are never clobbered.
