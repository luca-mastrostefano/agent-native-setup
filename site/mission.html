Compounding built modern software. Agent setups don't compound yet.

How tech growth became exponential

Technology grows exponentially for one reason: improvements compound. Nobody writes a compiler to ship a web app, or re-derives TCP to send a request. Every generation of engineers builds on the accumulated, shared work of the previous one — and the mechanism that made this possible is unglamorous: packaging. Package registries, open-source licenses, semantic versioning, dependency updates. The moment a solved problem could be published, discovered, adopted, and — crucially — kept up to date, solving it once was enough for everyone, forever. Each layer of shared infrastructure raised the floor the next layer was built on. That loop, repeated for decades, is the exponential.

Agent setups are stuck in the pre-packaging era

Coding agents are only as effective as the repo they work in: the contract they follow, the guardrails that catch their mistakes, the feedback loops that make quality compound instead of drift. Getting that setup right takes weeks of tuning prompts, hooks, review agents, and CI — and there is no one right answer: a Python backend, a TypeScript monorepo, and a legacy service each need a different one.

And today, all of that effort is private and perishable. Every team re-discovers its setup from scratch. Nothing is shared. What does get written down drifts the moment it lands. A brilliant reviewer prompt or a sharper quality gate dies in the one repo that had it. It's exactly where code was before package managers: everyone hand-copying snippets, no way to converge, every lesson re-learned one team over.

One good idea should propagate to every repo that needs it — not die in the repo that had it.

The mission: make agent setups compound

Our mission is to give agent setups the same loop that made code compound — as a community asset, open to everyone:

Package it. A setup is a profile — versioned, inspectable, forkable. Share it. A community index makes profiles searchable and ranked by real public usage, no telemetry. Converge. The best setup per use case wins on merit, gets forked, and improves — the way the best libraries do. Keep it alive. When the profile you adopted improves, one command flows the update into your repo without touching what you've customized. Your setup tracks the community's best current answer instead of freezing on the day you scaffolded it.

When the next team with your use case starts where you finished — not from zero — the weeks they save go into the next improvement, and that improvement flows back out. That's the compounding loop. It made software growth exponential. Agent-assisted engineering is the fastest-moving practice in software today, and it deserves the same engine.
